扩展

  • 全局成员
    • 浏览器中的js一般是指 ECMAScript + BOM + DOM
    • 而 Node 中除了提供了一些基本的全局成员外，其它都需要单独进行模块加载
    • dirname 和 filename
      • 一般用于操作文件或者其他路径操作的时候的绝对路径拼接
  • Node 中的模块系统
    • Node 中的模块系统是基于 CommonJS 的模块规范实现的
    • 一个js脚本文件就是一个模块
    • 所有代码都运行在模块作用域，不会污染全局
    • 使用 module.exports 作为模块的对外接口对象
    • 使用 require('模块标识') 方法加载模块
      • require() 方法指定的相对路径模块，和执行的时候所处的路径没有任何关系
      • 所以在使用 require() 方法的时候可以直接使用相对路径，没有问题
      • 参数以 ./ 或 ../ 开头的自定义文件模块
      • 参数不以 ./ 或 ../ 开头的 核心文件模块 或 第三方文件模块
    • 模块可以被多次加载，加载过的模块会被缓存
    • 模块是同步加载和执行的，加载模块时将运行模块文件中的每一行代码
